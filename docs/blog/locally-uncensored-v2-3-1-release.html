v2.3.1 — In-App Ollama Install, Configurable ComfyUI Port, and Provider Status Fix

Locally Uncensored v2.3.1 is a fast-follow to v2.3.0. Four targeted fixes: one-click Ollama install inside the app, configurable ComfyUI port and path, real install progress for ComfyUI, and provider status dots that actually reflect connection state.

In-App Ollama Download & Install

The onboarding wizard now has an Install Ollama button. Click it, and the app downloads, installs, starts, and auto-detects Ollama — zero manual steps.

A real-time progress bar shows download speed, bytes transferred, and an elapsed timer. The install runs silently in the background. Once complete, the app detects the running Ollama instance automatically and you’re ready to pull models.

Previously, users had to open a browser, download the Ollama installer, run it, wait, then come back to the app. Now it’s one click from inside the onboarding flow.

Configurable ComfyUI Port & Path

The ComfyUI port was hardcoded to 8188 in over 20 places across the codebase. That worked fine when everyone ran ComfyUI from source. It doesn’t work with the new ComfyUI Desktop App, which uses a different port.

v2.3.1 makes both the port and the path fully configurable in Settings > ComfyUI. If you’re running the ComfyUI Desktop App on a non-standard port, change it in settings and hit Connect. The app reconnects immediately.

This was reported as a bug 12 hours before we shipped the fix. Users with the ComfyUI Desktop App couldn’t connect at all — now they can.

ComfyUI Install Progress

The one-click ComfyUI install now shows step-by-step progress instead of getting stuck at “Starting…” forever.

  • Step 1/3: Clone — cloning the ComfyUI repository
  • Step 2/3: PyTorch — installing PyTorch with CUDA support
  • Step 3/3: Dependencies — installing remaining Python packages

Each step shows an elapsed timer so you know it’s not frozen. Previously the UI showed “Starting…” with no feedback for minutes during the PyTorch download. Users thought it was stuck and killed the process.

Provider Status Fix

The connection status dots in Settings now show actual status:

  • Green — connected and responding
  • Red — connection failed
  • Gray — unknown / not tested

Before this fix, every provider showed a green dot regardless of whether it was actually reachable. If Ollama wasn’t running, you’d still see green. That’s fixed — the dots now reflect real connection state.

Download

v2.3.1 is available now from GitHub Releases. Windows .exe and .msi installers are the most tested. Linux and macOS build from source.

If you’re using the ComfyUI Desktop App and couldn’t connect in v2.3.0, this update fixes it. Change your port in Settings > ComfyUI and you’re good.
